# 第02章 环境与项目结构

## 本章导读
稳定的测试结果来自稳定的工程环境。本章介绍项目依赖、目录布局及最小运行方式，为后续章节提供统一起点。

## 学习目标
- 理解项目中测试相关依赖的职责。
- 熟悉 Maven 标准目录与测试资源位置。
- 可以在本地独立运行全部测试。

## 2.1 基础环境
- JDK：1.8.0
- 构建工具：Maven
- 核心命令：`mvn test`

## 2.2 目录结构
```text
src/main/java         业务代码
src/test/java         测试代码
src/test/resources    测试资源（SQL、配置等）
pom.xml               依赖与插件配置
```

## 2.3 关键依赖说明
- `junit-jupiter`：JUnit 5 核心断言与测试模型。
- `mockito-junit-jupiter`：Mockito 与 JUnit 的集成扩展。
- `spring-boot-starter-test`：Spring 测试基础设施。
- `mybatis-spring-boot-starter-test`：MyBatis 测试切片支持。
- `h2`：测试用内存数据库。

## 2.4 运行与验证
首次进入项目后可直接执行：
```bash
mvn test
```
当输出 `BUILD SUCCESS` 且测试全部通过时，说明环境准备完成。

## 本章小结
在单元测试体系中，环境一致性与目录规范是最基础的工程纪律。

## 思考与练习
- 如果团队成员本地环境不一致，最容易导致哪些测试问题？
- 除了测试代码，`src/test/resources` 还应放哪些内容？
