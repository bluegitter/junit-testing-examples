# 第13章 测试报告与覆盖率报告生成

## 本章导读
单元测试执行通过只能说明“当前用例未失败”，但难以直观看出测试分布与覆盖情况。测试报告与覆盖率报告提供了可视化依据，是团队治理测试质量的重要工具。

## 学习目标
- 掌握一条命令生成测试报告与覆盖率报告。
- 理解各类报告文件的用途与读取方式。
- 能将报告产物接入 CI 流程。

## 13.1 一键生成命令
在项目根目录执行：

```bash
mvn clean verify
```

该命令会执行测试并在 `verify` 阶段生成报告。

## 13.2 报告产物说明
执行完成后可在 `target` 目录下查看：

- 单元测试原始报告（XML/TXT）
  - `target/surefire-reports/`
- 单元测试 HTML 报告
  - `target/site/surefire-report.html`
- 覆盖率 HTML 报告
  - `target/site/jacoco/index.html`
- 覆盖率 XML 报告（便于 CI/质量平台读取）
  - `target/site/jacoco/jacoco.xml`

## 13.3 如何阅读单元测试报告
单元测试报告重点关注：
- `Tests run`：总执行数量。
- `Failures`：断言失败数量。
- `Errors`：运行期异常数量。
- `Skipped`：跳过数量。

建议优先清零 `Failures` 和 `Errors`，并审查 `Skipped` 是否合理。

## 13.4 如何阅读覆盖率报告
JaCoCo 报告常见维度：
- Instructions：指令覆盖率。
- Branches：分支覆盖率。
- Lines：行覆盖率。
- Methods / Classes：方法与类覆盖率。

实践中应优先关注：
- 核心业务类覆盖是否不足。
- 异常分支与边界分支是否遗漏。
- 覆盖率下降是否与本次改动相关。

## 13.5 与 CI 集成建议
- 在 CI 任务中固定执行 `mvn clean verify`。
- 将 `target/site/jacoco/jacoco.xml` 上传到质量平台（如 SonarQube）。
- 配置最小覆盖率阈值，覆盖率异常下降时阻断合并。

## 本章小结
测试报告回答“有没有通过”，覆盖率报告回答“测得够不够”。两者结合，才能持续提升测试质量。

## 思考与练习
- 执行一次 `mvn clean verify`，记录当前分支覆盖率。
- 选择一个覆盖率较低的类，补充 2 个边界用例并对比报告变化。
